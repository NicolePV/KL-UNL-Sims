Note that T is always rounded off to the nearest integer value while the precision with which L is defined varies depending on its amplitude, and the calculated radius value is always shown to up to four decimal places.

We are using MathJax v3, which is configured to automatically convert source LaTeX into semantic, screen-reader-accessible MathML. It includes explicit ARIA live regions and semantic form fields to support both visual users and screen reader users, meeting WCAG AA standards. When the DOM changes dynamically via JavaScript, some screen readers fail to re-announce the newly generated MathML. To solve this issue, the rendered math container is hidden from assistive tech via aria-hidden="true", and a companion hidden div using aria-live="polite" is defined to handle verbal notifications.
